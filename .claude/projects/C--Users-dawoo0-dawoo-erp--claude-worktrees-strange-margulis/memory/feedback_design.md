type: feedback
---

"디자인 망했네 디자인제외하고 업무지시사항만해줘 디자인 따로잡자" — 디자인 토큰 일괄 교체 작업 후.
**Why:** 사장님이 직접 디자인을 잡을 예정. 개발자가 임의로 디자인을 만지면 오히려 망가짐.
**How to apply:** 디자인(CSS 클래스, 색상, 폰트, 간격, 레이아웃) 절대 임의 수정 금지. 사장님이 명시적으로 디자인 변경을 요청할 때만 수정. 새 컴포넌트 만들 때는 기존 페이지의 스타일을 그대로 복사해서 사용.

"디자인 참 알록달록하네" — 직원관리 페이지가 컬러풀한 카드 그리드였을 때 거부.
**Why:** 업무용 도구는 깔끔하고 절제된 디자인이어야 함.
**How to apply:** 테이블 기반 레이아웃 우선, 상태 뱃지만 컬러.

"입력 필드를 항상 보여주지 마" — 읽기 모드 기본, 클릭 시 인라인 편집.
**Why:** 네모 테두리 input 박스 남발하면 지저분해 보임.
**How to apply:** 평소에는 텍스트로 표시, 클릭하면 그 자리에서 수정 가능한 인라인 편집 방식.

"이렇게 대충 만들지말고" — 등록 폼은 실무 필드 전부 포함해야 함.
