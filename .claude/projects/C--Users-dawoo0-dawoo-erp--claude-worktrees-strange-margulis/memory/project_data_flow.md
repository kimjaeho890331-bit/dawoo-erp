## 1. 캘린더 데이터 흐름

**DB: schedules 테이블 1개**

입력 4곳:
- 접수대장 → 실측/시공/제출일
- 현장관리 → 착공~준공 공정
- 직원/관리자 → 개인 일정 등록
- AI 비서 → 홍보/미팅 일정 제안

출력 4곳:
- 업무 캘린더: 지원사업 + 개인일정 + 홍보. 필터: 직원별/종류별
- 현장 캘린더: 현장별 공정 + 협력업체. 체크박스로 현장 켜기/끄기
- 접수대장 상세: 해당 건의 일정만 표시
- 현장 상세(아코디언): 해당 현장 캘린더 고정

AI 캘린더 기능 4가지:
1. 빈 일정 감지 → 홍보/미팅 제안
2. 일정 충돌 체크 → 협력업체 겹침 알림
3. 공문 패턴 예측 → 작년 기준 2달 전 알림
4. 마감 알림 → 제출 D-day

→ 업무 캘린더 하단에 AI 제안 카드 고정 표시

**Why:** 데이터 하나, 뷰 여러 개 원칙. schedules 1개 테이블로 통일.
**How to apply:** 새 일정 입력 시 항상 schedules 테이블에 schedule_type 구분해서 저장. 각 화면에서는 필터링만.

## 2. 보고서/분석 데이터 흐름

12개 소스 → AI 분석 엔진 → 4종 보고서 → 총괄 보고서 → 관리자 분석 + KPI

**12개 소스:**
- 접수대장 (접수/완료 건수)
- 현장관리 (공정률/일지/사진)
- 캘린더 (일정 준수율)
- 활동 로그 (직원별 시스템 사용)
- 지출결의서 (현장별/항목별 지출)
- 카드지출 (직원별 CSV 분석)
- 고정지출 (매월 고정비)
- 거래처 DB (협력업체 평가)
- 홍보현황 (지역별 방문 기록)
- A/S 관리 (하자 패턴/빈도)
- 미수금 (완료 건 기준)
- 현장 보고서 (소장 미팅 후 최종)

**4종 보고서:**
- 매일: 직원 활동 요약
- 주간: 지원사업 현황 + 현장 공정 진척 + 직원 성과/미수금
- 월간: 카드/고정지출 분석 + KPI 종합 + 지원사업 통계
- 현장 마감: 최종 보고서 + 수익률 분석 + 업체 평가

**총괄 보고서 (주간/월간):**
긴급 → 주의 → 정상 우선순위 정렬

**출력:**
- 관리자 분석 페이지: 5개 탭 (김재호 전용)
- KPI 페이지: 직원/현장/업체 성과

**Why:** 모든 데이터가 보고서/KPI로 모이는 구조. 사장님이 숫자만 보는 게 아니라 상황 설명이 포함된 글 형태 보고서.
**How to apply:** 각 페이지에서 데이터 입력 시 activity_log 자동 기록. 보고서 생성 시 12개 소스 전부 조회해서 AI가 글로 작성.
