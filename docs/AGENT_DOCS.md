# docs/AGENT_DOCS.md - 서류팀 에이전트

## 역할
- PDF 서류 자동 생성 (폼필드 방식)
- 견적서 관리 (웹 스프레드시트)
- 서류함 템플릿 관리
- OCR (통장사본 → 은행/계좌/예금주)

---

## 도구 (Tools)

| 도구 | 용도 |
|------|------|
| templates 테이블 | 서류함에서 템플릿 조회 |
| pypdf | 폼필드에 데이터 자동 입력 |
| pdfplumber | 좌표 추출 (신규 템플릿 분석) |
| Supabase Storage | PDF/첨부파일 저장 |
| documents 테이블 | 생성된 서류 등록 |
| OCR | 통장사본 → 은행/계좌/예금주 추출 |

---

## 서류 생성 흐름

### 신청서
```
"분당 래미안 신청서 만들어줘"
    ↓
projects에서 해당 프로젝트 조회
    ↓
시(city) 확인 → 서류함에서 해당 시 신청서 템플릿 가져옴
    ↓
데이터 충분? 
  YES → 폼필드에 자동 입력 → PDF 생성 → Storage 저장
  NO → "XX 부족합니다" 차단 + 구체적 부족 항목 안내
    ↓
documents 테이블에 등록 + 접수대장 서류탭 연동
activity_log 기록
```

### 견적서
```
접수대장 1단계 → "견적서 열기"
    ↓
서류함에서 견적서 기본폼(웹 스프레드시트) 불러옴
    ↓
고객 데이터 자동 입력 (빌라명, 주소, 소유주 등)
    ↓
사람이 입력: 실측 사이즈 + 공사종류 체크
    ↓
실시간 자동 계산
    ↓
금액 조정 필요시 → 사람이 단가 직접 수정
    ↓
저장 → Storage + estimates 테이블
PDF/엑셀 출력 가능
```

### 완료보고서
```
공사 완료 후 → AI가 프로젝트 데이터 기반 초안 생성
    ↓
직원이 확인/수정
    ↓
PDF 생성 → Storage 저장
```

---

## OCR 흐름 (통장사본)

```
접수대장 기본정보 → 통장사본 드래그앤드롭
    ↓
OCR 분석: 은행명, 계좌번호, 예금주 추출
    ↓
projects 테이블에 자동 입력
    ↓
신청서 PDF에 통장번호 자동 반영
```

---

## 서류 부족 시 차단 규칙

| 서류 | 필수 데이터 | 없으면 |
|------|-----------|--------|
| 신청서 | 빌라명, 주소, 소유주, 연락처, 면적, 세대수 | 생성 차단 + 안내 |
| 견적서 | 실측 사이즈, 공사종류 | 계산 불가 안내 |
| 완료보고서 | 공사 완료일, 시공전/후 사진 | 생성 차단 + 안내 |
| 착공계 | 승인일, 시공일, 시공업체 | 생성 차단 + 안내 |

---

## 신규 템플릿 등록 흐름

```
재호가 서류함에 새 양식 PDF 올림
    ↓
pdfplumber로 텍스트 좌표 분석
    ↓
빈칸 위치에 폼필드 자동 추가 (제목 기반 매칭)
    ↓
재호 확인 → 수정 → 최종 확정
    ↓
templates 테이블 + Storage에 저장
```
