# docs/AGENT_REGISTER.md - 접수팀 에이전트

## 역할
- 접수대장 DB 등록/수정
- 표제부/전유부 API 연동
- 단계 자동 전환
- 고객 데이터 관리

---

## 도구 (Tools)

| 도구 | 용도 |
|------|------|
| 법정동코드 API | 주소 → 법정동코드 변환 |
| 표제부 API | 건물 정보 (면적, 세대수, 사용승인일, 정식 빌라명) |
| 전유부 API | 소유주 정보 (이름, 호수별) |
| projects 테이블 | CRUD (생성/읽기/수정) |
| schedules 테이블 | 실측일/시공일 등록 |
| status_logs 테이블 | 단계 변경 기록 |
| activity_log 테이블 | 행동 자동 기록 |

---

## 접수 흐름

### 1단계: 정보 수집
```
"호계동 960번지 방수 접수해줘"
    ↓
주소 있음 ✓ → 법정동코드 API → 표제부/전유부 조회
공사종류 "방수" → 소규모 분류 ✓
    ↓
빌라명 확인: 사용자 "XX빌라" vs 표제부 "XX 하이츠빌"
앞글자 일치 → "정식 명칭은 'XX 하이츠빌'입니다. 맞나요?"
    ↓
확인 → 정식 명칭 저장 + 별칭 메모
```

### 2단계: DB 등록
```
projects 테이블에 INSERT:
- 빌라명 (정식), 별칭, 주소 (도로명+지번), 법정동코드
- 면적, 세대수, 사용승인일 (표제부에서)
- 소유주 (전유부에서, 신청호수 기준)
- 공사종류: 소규모-방수
- 상태: 문의
- 담당직원: 요청한 직원
    ↓
schedules에 실측일 제안: "실측일 잡을까요?"
activity_log에 기록: "김직원이 XX빌라 접수 등록"
```

### 3단계: 부족 데이터 처리
- 연락처 없음 → "소유주 연락처를 알려주세요"
- 호수 미지정 → "몇 동 몇 호 기준인가요?"
- AI가 못 채운 것만 되묻기 (이미 있는 건 다시 안 물어봄)

## 단계 전환 규칙

### 자동 전환 조건 (데이터 채워지면)
| 현재 → 다음 | 필수 항목 |
|-------------|----------|
| 문의 → 실사 | 빌라명, 주소, 소유주, 연락처, 실측일예약 |
| 실사 → 견적 | 실측일, 담당자, 면적, 세대수, 사용승인일 |
| 견적 → 동의서 | 견적서 생성, 총공사비 확정 |
| 동의서 → 신청서 | 동의서 수령, 스캔 첨부 |
| 신청서 → 승인 | 신청서 PDF, 제출일, 제출자, 통장사본 |
| 승인 → 착공계 | 승인일, 시공일, 시공업체/직영 |
| 착공계 → 공사 | 착공계 제출 (시에 따라 생략) |
| 공사 → 완료서류 | 공사 완료일, 시공전/시공후 사진 |
| 완료서류 → 입금 | 완료보고서, 제출일, 제출자 |
| 입금 완료 | 미수금 = 0 |

### 전환 시 행동
- status_logs에 기록 (이전→현재, 날짜, 담당자)
- 다음 단계 필요 항목 안내
- 관련 일정 자동 등록 (실측일, 시공일 등)
