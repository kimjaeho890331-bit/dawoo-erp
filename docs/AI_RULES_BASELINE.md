# AI 기준선 + 학습 + 기억 구조

> AI가 어떻게 기준을 만들고, 학습하고, 기억하는지의 기술 설계.

---

## 1. 자동 기준선 생성

대표가 기준을 일일이 설정하지 않는다. AI가 데이터를 보고 자동 생성.

### 데이터 축적 기간별 AI 행동

| 기간 | 모드 | 행동 |
|------|------|------|
| 0~1개월 | 학습 모드 | 기준 없음, 데이터만 수집 |
| 1~3개월 | 초기 기준 | 3개월 평균으로 기준선 자동 설정, 대표에게 확인 요청 |
| 3개월~ | 운영 모드 | 3개월 이동평균으로 기준선 자동 갱신, 이상치 자동 알림 |

### 자동 생성되는 기준선 목록

**프로젝트 기준선**
- 접수~실측 평균 소요일 → 150% 초과 시 알림
- 실측~견적 평균 소요일 → 150% 초과 시 알림
- 공사종류별 평균 마진율 → 5%p 이하 이탈 시 알림
- 공사종류별 평균 공기 → 120% 초과 시 알림

**직원 기준선**
- 월 평균 접수 건수 → 50% 미만 시 알림
- 월 평균 홍보 횟수 → 0건이면 알림
- 일지 작성률 → 80% 미만 시 알림
- 일지 평균 글자수 → 기준선 50% 미만 시 코칭

**재무 기준선**
- 현장별 월 지출 평균 → 130% 초과 시 알림
- 고정비 월 평균 → 20% 급증 시 알림
- 미수금 평균 회수일 → 60일 초과 건 알림

### 알림 수준

| 수준 | 기준 | 대상 |
|------|------|------|
| 정상 | 기준선 ±20% 이내 | 알림 없음 |
| 주의 | 기준선 ±20~50% | 대표 주간보고에 포함 |
| 경고 | 기준선 50% 이상 이탈 | 대표 즉시 알림 |
| 긴급 | 기한 초과/미수금 60일 등 | 대표 + 담당 직원 |

## 2. 대표 직접 설정 (최소한만)

AI가 못 만드는 기준 = 대표의 경영 판단이 필요한 것.

### KPI 설정 페이지에서 1번만 설정

| 항목 | 기본값 | 비고 |
|------|--------|------|
| 마진율 목표 | 30% | 수정 가능 |
| 미수금 경고 기준 | 60일 | 대화로 변경 가능 |
| 직원 월 최소 홍보 | 4회 | 대화로 변경 가능 |
| 일지 최소 글자수 | 100자 | 대화로 변경 가능 |
| 정성평가 항목별 배점 | 기본값 제공 | KPI 고도화 시 확정 |

이후 AI가 **설정 + 자동 기준선을 조합**해서 판단.

## 3. 시기 패턴 학습 (1년 이상 데이터 축적 후)

| 패턴 | AI 행동 |
|------|--------|
| "작년 4월 소규모 접수 15건, 올해 8건" | "홍보 강화 고려해보세요" |
| "매년 7월 방수 A/S 집중" | "6월에 방수 현장 점검 권장" |
| "매년 1월 수도 동파 문의 집중" | "12월에 기존 고객 점검 안내 제안" |

**기술 구현**: Edge Function cron이 매주 데이터 집계 → ai_knowledge 테이블에 저장

## 4. AI 기억 구조 (대화로 수정 가능)

### 대화로 되는 것 (ai_memory에 저장)

| 예시 | 저장 내용 |
|------|----------|
| "보고서에 미수금 금액 큰 순으로" | ceo_preference: 미수금 보고 금액 내림차순 |
| "미수금 경고 45일로 해줘" | ai_knowledge 수정: 미수금 경고 45일 |
| "보고서 더 짧게" | ceo_preference: 보고서 간결하게 |
| "방수는 무조건 사진 8장" | rule: 방수공사 사진 최소 8장 |
| "OO건설 C등급은 우천 때문" | vendor_note: OO건설 우천 감안 |

### 코드 수정이 필요한 것

| 예시 | 이유 |
|------|------|
| "카카오톡으로도 알림 보내줘" | 새 기능 추가 |
| "캘린더에 날씨 넣어줘" | UI 변경 |
| "국세청 API 연동해줘" | 새 데이터 소스 |

### 구분 기준

AI가 "어떻게" 행동하는지 (판단 기준, 보고 형식, 선호도)
→ 대화로 수정 가능

AI가 "무엇을" 할 수 있는지 (새 기능, 새 연동)
→ 코드 수정 필요

## 5. 매 대화 시 AI가 하는 것

읽기: ai_memory + ai_knowledge + chat_history (최근 20건)
→ "이 사람의 선호, 현재 기준선, 최근 맥락"

대화: 사용자 질문에 답변

판단: "이 대화에서 기억할 게 있나?"
→ 있으면 ai_memory에 자동 저장
→ 기준선 변경이면 ai_knowledge 수정

저장: 대화 기록 → chat_history
