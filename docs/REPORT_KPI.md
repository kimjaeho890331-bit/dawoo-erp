## 2. 보고서 종류 (4종)

### 2-1. 매일 보고서
- 자동 생성 (매일 오후 5시)
- 내용:
  - 직원별 활동 요약 (활동 로그 기반)
  - 긴급 알림 (서류 마감 D-3 이하, 미수금 30일 초과, 공정 지연)
  - 오늘 등록된 접수 건수
  - 오늘 변경된 상태 목록

### 2-2. 주간 보고서
- 자동 생성 (매주 월요일)
- 내용:
  - **지원사업**: 수도 — 이번 주 접수 몇 건, 옥내/공용 각 몇 건 완료. 소규모 — 접수/완료 건수. 1년 스케줄 기준 진행률.
  - **현장**: 주간 공정 관리 (계획 대비 실제 진척), 현장별 이슈
  - **직원**: 활동 로그 기반 성과 (접수 N건, 실측 N건, 서류 N건)
  - **미수금**: 공사 완료 건 기준, 30일 이상 미수금 목록
  - **홍보**: 이번 주 방문 지역, 미방문 지역 현황
  - **주간 통합 요약** (위 내용 AI가 종합)

### 2-3. 월간 보고서
- 자동 생성 (매월 1일)
- 내용:
  - **지원사업**: 월간 통계 (시별, 공사종류별, 전월 대비)
  - **지출**: 카드지출 + 고정지출 — 새는 돈 찾기 목적
    - 건당 기준 이상치 (식대 15,000원+, 편의점 매일 패턴)
    - 고정지출 변동분
  - **KPI 종합**: 직원별, 현장별 (아래 기준표 참조)
  - **월간 통합 요약**

### 2-4. 현장 마감 보고서
- 현장 완료 시 생성
- 내용:
  - **수익률**: 마감 기준 (월별 정산 아닌, 현장 끝났을 때 기준)
  - **견적 대비 실제**: 견적 금액 vs 실제 지출 차이 분석
  - **협력업체 평가**: 납기 준수, 시공 품질, A/S 발생
  - **소장 소명**: AI가 초안 → 소장이 의견 추가 → 관리자 미팅 → 최종 확정
  - 프로세스:
    1. AI가 현장일지 기반 초안 자동 생성
    2. 소장이 본인 의견 + 소명 추가 (견적 오차 원인, 공기 지연 사유 등)
    3. 관리자(재호)와 소장 미팅 → 수정/보완
    4. 최종 보고서 확정 → KPI 반영

## 3. 총괄 AI (보고서 정리 AI)

> 재호 요청: "보고서가 10개 넘어오면 너무 많으니 10개를 관리해서 나한테 다시 보고서를 만들어줄 AI가 필요하다"

### 역할
- 모든 보고서를 읽고 → 우선순위로 정렬 → 1장 요약이 아니라 "읽는 순서를 잡아주는" 것

### 정렬 기준
1. **긴급** (빨강) — 공기 지연, 미수금 60일+, 서류 마감 D-1, 안전사고, 이상 지출
2. **주의** (노랑) — 미수금 30일+, 공정 5일+ 지연, KPI 하락, 미방문 90일+
3. **정상** (초록) — 이상 없음, 정상 진행

### 화면 구성 (관리자 분석 페이지)

**5개 탭:**

1. **긴급/요약 탭** (열면 가장 먼저 보이는 것)
   - 우선순위: AI 긴급사항 → 사람(직원) → 현장 → 돈
   - 긴급 카드: 빨간 배경, 즉시 행동 필요
   - 주의 카드: 노란 배경, 확인 필요
   - 정상: "이상 없음" 한 줄
   - 미확인 보고서 상단 고정, 확인하면 아래로

2. **직원 성과 탭**
   - 직원별 활동량 바 차트
   - 접수/실측/시공/서류/홍보 건수
   - AI 분석 코멘트 ("김직원 이번 주 접수 3건, 전주 대비 50% 증가")
   - 활동 로그 상세 열람 (관리자만)

3. **현장 현황 탭**
   - 현장별 공정률, 예산/지출률
   - 주간 상태 변화
   - 이슈 있는 현장 상단

4. **매출/지출 탭**
   - 숫자 요약: 총 매출, 총 지출, 미수금 합계
   - AI 지출 분석: 새는 돈 감지 결과
   - 미수금: 완료 건 기준, 오래된 순
   - 현장 수익률: 마감 기준

5. **보고서 목록 탭**
   - 매일/주간/월간/현장마감 보고서 전체 목록
   - 미확인 상단 고정
   - 클릭하면 상세 열람
   - 기간 필터 (이번 주, 이번 달, 전체)

## 4. KPI 평가 기준표

### 4-1. 총 배점 구조

| 영역 | 배점 | 평가 방식 |
|------|------|----------|
| 지원사업 성과 | 30점 | 정량 (AI 자동) |
| 현장 성과 | 25점 | 정량 (AI 자동) |
| 업무 효율 | 25점 | 정량 (AI) + 정성 (대표) |
| 협력업체/외부 | 10점 | 정량 (AI 자동) |
| 업무 상황점수 | 10점 | 정성 (대표 설정) |
| **합계** | **100점** | |

### 4-2. 지원사업 성과 (30점) — 직원별

| 지표 | 배점 | 우수 (100%) | 보통 (70%) | 개선 (40%) | 소스 |
|------|------|-----------|-----------|-----------|------|
| 접수 전환율 (문의→완료) | 10점 | 70%+ | 50~69% | 50%- | projects |
| 일정 준수율 (마감 내 제출) | 8점 | 90%+ | 80~89% | 80%- | schedules, projects |
| 서류 오류율 | 6점 | 5%- | 5~10% | 10%+ | documents, 반려 기록 |
| 미수금 회수 기간 | 6점 | 30일- | 30~60일 | 60일+ | payments |

**산출 예시**: 접수 전환율 75% → 10점×100% = 10점 / 일정 준수율 85% → 8점×70% = 5.6점

### 4-3. 현장 성과 (25점) — 현장별/소장별

| 지표 | 배점 | 우수 (100%) | 보통 (70%) | 개선 (40%) | 소스 |
|------|------|-----------|-----------|-----------|------|
| 공정 준수율 (계획 대비 실제) | 8점 | 95%+ | 85~94% | 85%- | site_tasks, site_logs |
| 예산 대비 지출 | 7점 | 100%- | 100~110% | 110%+ | sites, expenses |
| 하자 발생 건수 | 5점 | 0건 | 1~2건 | 3건+ | as_records |
| 현장일지 성실도 | 5점 | 매일 작성 | 주 3회+ | 주 3회- | site_logs |

### 4-4. 업무 효율 (25점) — 직원별

| 지표 | 배점 | 평가방식 | 우수 | 보통 | 개선 | 소스 |
|------|------|---------|------|------|------|------|
| 활동량 추이 | 8점 | 정량 | 본인 전월 대비 유지/증가 | 10% 감소 이내 | 10%+ 감소 | activity_log |
| 홍보 달성률 | 7점 | 정량 | 월 목표 80%+ | 60~79% | 60%- | promo_records |
| 캘린더 빈 일정 비율 | 5점 | 정량 (참고) | 20%- | 20~40% | 40%+ | schedules |
| 업무 태도/협업 | 5점 | 정성 | 대표 평가 | | | 대표 직접 입력 |

**활동량 추이 기준**: 본인 과거 대비 비교 (직원 간 비교 아님). 사무직처럼 일정이 안 올라가는 경우 → 시스템 사용 로그(접수등록, 서류생성, 메모 등)가 업무 증명.

### 4-5. 협력업체/일용직 평가 (10점)

| 지표 | 배점 | 우수 | 보통 | 개선 | 소스 |
|------|------|------|------|------|------|
| 납기 준수율 | 4점 | 95%+ | 85~94% | 85%- | site_tasks |
| 시공 품질 (A/S 빈도) | 3점 | 0건 | 1건 | 2건+ | as_records |
| 재고용 횟수 | 3점 | 5회+ | 2~4회 | 1회 이하 | vendors |

### 4-6. 업무 상황점수 (10점) — 대표 설정

> 재호 요청: "정량으로 안 잡히는 업무들도 점수를 줄 수 있게"

이 영역은 AI가 자동 산출하지 않고 **대표가 직접 배점**한다.

| 항목 | 배점 | 설명 |
|------|------|------|
| 긴급 대응력 | 3점 | 긴급 상황(누수, 민원, 시청 요청)에 빠르게 대응했는가 |
| 팀 기여도 | 3점 | 다른 직원/현장 도움, 정보 공유, 협업 |
| 자기 개발 | 2점 | 새로운 업무 학습, 프로세스 개선 제안 |
| 고객 관리 | 2점 | 고객 만족도, 클레임 대응, 재계약 유도 |

**평가 방식**: 대표가 분기/반기별로 직접 입력 (5단계: 탁월/우수/보통/미흡/부족)
**탁월=100%, 우수=80%, 보통=60%, 미흡=40%, 부족=20%**

## 5. KPI 전체 등급표

| 총점 | 등급 | 의미 |
|------|------|------|
| 90~100 | S | 탁월 — 핵심 인재 |
| 80~89 | A | 우수 — 기대 이상 |
| 70~79 | B | 양호 — 기대 충족 |
| 60~69 | C | 보통 — 개선 여지 |
| 60 미만 | D | 미흡 — 면담 필요 |

## 6. 정량 vs 정성 요약

| 구분 | 비율 | 평가자 | 특징 |
|------|------|--------|------|
| 정량 | 75점 | AI 자동 | 데이터 기반, 객관적, 매달 자동 산출 |
| 정성 | 25점 | 대표 직접 | 업무태도+상황점수, 분기별 입력 |

정량은 AI가 매달 자동으로 산출 → 직원도 본인 점수 확인 가능 (동기부여)
정성은 대표가 면담/관찰 기반으로 분기별 입력

## 7. KPI 전문 에이전트

- 분석 페이지 안에서 KPI 기준 수정/보완 가능
- "접수 전환율 우수 기준을 75%로 올려줘" → 즉시 반영
- 분기별 기준 리뷰 제안 ("이번 분기 전체 평균 접수전환율 68%, 우수 기준 70%는 적절합니다")
- 직원별 KPI 추이 그래프 자동 생성

---

## 8. 직원이 보는 KPI

직원 대시보드에서 본인 KPI만 확인 가능:
- 내 현재 점수 / 등급
- 영역별 점수 바 차트
- 전월 대비 변화 (화살표)
- AI 코멘트 ("접수 전환율이 올랐어요. 이번 달 실측 일정을 더 잡으면 A등급 가능합니다")
- 다른 직원 점수는 안 보임

## 9. 데이터 흐름 요약

```
12개 소스 → AI 분석 → 4종 보고서 (매일/주간/월간/현장마감)
                    ↓
              총괄 AI가 전체 취합
                    ↓
          관리자 분석 페이지 (5탭)
          긴급 → 주의 → 정상 순 정렬
          미확인 보고서 상단 고정
                    ↓
          KPI 자동 산출 (75점 정량)
          + 대표 직접 입력 (25점 정성)
                    ↓
          직원 대시보드 (본인 KPI만)
```

## 10. 지출 이상치 감지 기준

| 항목 | 기준 | 플래그 |
|------|------|--------|
| 식대 | 1건 15,000원+ | 노랑 경고 |
| 편의점 | 매일 4,000원 패턴 | 노랑 경고 |
| 유류비 | 1건 100,000원+ | 빨강 경고 |
| 자재비 | 견적 대비 120%+ | 빨강 경고 |
| 건당 기준 | 합산 아님, 건당으로 판단 | |
| 현장 수익률 | 월별 아닌 마감 기준 | |
