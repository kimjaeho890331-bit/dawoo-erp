## 1. 배경

- **DAWOO ERP**: 정부 지원사업(수도/소규모) 접수~수금 + 건설 현장관리. Next.js 16 + Supabase(PostgreSQL·Auth·Storage) + Claude + Vercel.
- **인원 6명**, 그중 **ERP 관리권한 2명**: 김재호 이사, 김태정 대리. 나머지 직원은 조회 + 단순 자동화 사용.
- **현재 접수**: 유선상담 → 카카오톡으로 고객정보 수신 → 담당자가 ERP 수동 입력.
- **현재 AI**: 인앱 AI 비서(`/api/chat`) + 텔레그램 봇. 공용 코어(`payments.ts`) 위 멀티채널.

## 2. 목표

1. ERP 자동화를 Hermes(Nous `hermes-agent`)로 구축한다.
2. **반복 입력·서류 작성·보고를 AI가**, 사람은 **상담·판단·확인**에 집중.
3. 자동화 가능 지점: 접수 등록, 외부 API 자동조회, **공문 자동생성**, 입금 처리, 보고 4종, 문지기 알림, 자가학습.

## 5. 컴포넌트와 책임

### 5.1 채널 (2계층)
- **텔레그램 '고객접수' 단톡방**: 담당자가 카톡 고객정보(텍스트/스크린샷)를 투입 → Hermes 봇이 읽어 자동등록. 봇은 privacy mode off / 방 관리자. 방은 **비공개·최소인원**.
- **Slack(관리자 2명)**: Hermes에 지시·에러처리·승인·리포트 수신. 스레드·블록·버튼으로 풍부한 통제.
- 직원은 새 도구 학습 0(이미 텔레그램 사용). 관리자만 Slack 학습.

### 5.2 Hermes Agent (두뇌, 매니지드 클라우드)
- 게이트웨이(텔레그램+Slack), 에이전트 루프(Claude), cron(보고·알림), llm-wiki 자가학습, 문서엔진(공문).
- **매니지드 호스팅**이라 OS·재기동·백업·확장은 플랫폼이 담당(운영부담 최소 — 관리 인력 2명 고려).

### 5.3 ERP MCP 서버
- 기존 도구 핸들러 재사용: `search_address`/`get_building_info`/`get_unit_info`/`register_project`/`search_projects`/입금(`applyDepositAndAdvanceStatus`) + 신규 `update_project`/`update_status`/`manage_schedule`/`get_dashboard_stats`/공문생성.
- **인증**: 정적 god 토큰 금지. **스코프 토큰 + 호출별 감사로그 + Slack/텔레그램 사용자↔`staff.role` 권한 체크**. 쓰기는 관리자 승인 게이트.
- 저장 전 `validateProjectData()` 유지.

### 5.4 ERP (화면·데이터·대시보드)
- 화면·Supabase 유지. **관리자 자동화 대시보드 신설**(§9).
- 인앱 AI 비서: **병행 유지**, 제거는 P-말기 데이터 기반 결정.

### 5.5 Obsidian = llm-wiki (지식, PII 제로)
- 구조: `SCHEMA.md`/`index.md`/`log.md`/`raw/`/`entities/`/`concepts/`/`comparisons/`/`queries/`.
- **PII 제로 원칙**: 고객 이름·연락처·주소를 위키에 쓰지 않음. 대신 **프로젝트 ID(익명 참조키)** 로 링크 → 실제 PII는 Supabase에서 조회.
- **2계층 경계**: 위키엔 *판단·평가·노하우 + 참조키*만, **숫자·잔액·일정은 DB**.
- **동기화**: 라이브 위키는 **Obsidian Sync(headless)** 또는 "에이전트가 쓰고 사람은 대화로 큐레이션"(동시 파일편집 금지). 원시 git 양방향 지양.

### 5.6 문서 엔진 (공문) — `hwpkit` (PoC 검증 완료)
- **엔진 = `hwpkit`**(오픈소스 MIT, 순수 파이썬). 실제 수원 동의서로 **읽기 → 필드 주입(`inject_text(index, text)`) → 저장 → 재확인** 성공. 한컴·윈도우·COM·LibreOffice·변환 **전부 불필요** — Hermes와 같은 리눅스 클라우드에서 동작.
- `.hwp`(바이너리)·`.hwpx` 둘 다 동일 API. `inject_text`(빈칸 채우기), `replace_text`, `place_image`(도장 찍기). Hermes가 ERP 데이터 → 칸 index를 **양식별 1회 매핑**(§7).
- 대안 라이브러리: `python-hwpx`(HWPX 전용, `fill_by_path`).
- **디커플**: 메인 자동화(접수·입금·리포트)는 이 트랙에 의존하지 않음.

## 6. 핵심 데이터 흐름

### Flow 1 — 접수 자동화
담당자 유선상담 → 카톡 수신 → **텔레그램 접수방 투입** → Hermes 파싱(스크린샷이면 Vision) → `search_address`→`get_building_info`→`get_unit_info`→`register_project` 자동등록·외부조회 → 방에 결과 회신 → 관리자 확인. (대부분 기존 파이프라인 재사용)

### Flow 2 — 공문 자동생성 (지역×연도 변동성 §7)
단계 전환 시(신청서제출/착공계/완료서류) Hermes가 `해당 시·연도 템플릿(HWPX) + 연도규칙 + 프로젝트/견적 데이터`로 공문 채움 → 관리자 승인 → 드라이브 저장. **자동제출 금지.**

### Flow 3 — 입금 처리
입금 문자 투입 → `match_deposit`→후보확인→`record_deposit`→`applyDepositAndAdvanceStatus`(중복체크·재계산·단계전환·로그). 기존 `payments.ts` 재사용 + **멱등키·dry-run 프리뷰**.

### Flow 4 — 리포트/알림 (cron)
보고 4종(긴급 즉시·주간 월요일·월간 1일·현장마감) + 문지기 알림(서류누락·미수금 D+·공정지연) → Slack(관리자)/텔레그램.

### Flow 5 — 자가학습 (llm-wiki)
업무 발생 → Hermes가 *판단·노하우 + 참조키*를 위키에 ingest(PII 제로) → 관리자 큐레이션 → query로 활용 → 정기 lint(모순·낡음·고아).

## 7. 공문 지역×연도 변동성 해결 (유지)

원칙: 변하는 지식은 사람이 다듬는 Obsidian에, 엔진은 그대로.
- 템플릿(시×문서×연도) 버전 보관 + Hermes 의미기반 분석
- 연도별 규칙(지원률·자격·단가)은 위키 `concepts/지원사업/{연도}/{시}.md`(프론트매터)
- 변경 시 Hermes diff → 사람은 변경분만 확인 / 변경 없으면 자동 재사용
- 안전: 초안+근거 제시 → 사람 승인, 자동제출 금지

## 8. 2계층 기억 원칙

| 저장소 | 담는 것 | 규칙 |

## 9. 관리자 자동화 대시보드 (관측가능성)

- **데이터**: `agent_actions` 감사테이블(시각·행위·대상·입력원본·결과·신뢰도·상태·전후값). MCP 도구가 호출 시 자동 기록. + 기존 `status_logs`/`activity_log`.
- **화면 6블록**: ① 오늘 요약 ② 승인 대기 큐 ③ 감사 로그 ④ **원본↔결과 대조**(텔레그램 원문 ↔ 등록 필드) ⑤ 이상/실패 ⑥ 자가학습 현황.
- **구축**: `src/app/admin/automation/page.tsx` + 컴포넌트, **RLS로 관리자 2명만**, realtime 구독. 새 테이블은 `supabase-migration` 스킬.
- **푸시**: 일일 다이제스트 + 실시간 이상알림 → Slack(관리자). = 풀(대시보드)+푸시(Slack).

## 10. 보안

- ERP MCP: **스코프 토큰 + 감사로그**, `service_role` 직노출 금지, 쓰기=관리자 승인.
- 권한: 관리자 2명만 쓰기/관리. 직원은 조회+단순 자동화.
- Obsidian: **PII 제로**, private 저장소.
- 텔레그램 접수방: 비공개·최소인원(PII는 즉시 Supabase로, 방은 전송 통로).
- 공문 **자동제출 금지**, 사람 승인.
- 쓰기 도구: **멱등키·dry-run 프리뷰·중복체크**로 garbage-in 완화. 저신뢰 파싱은 Hermes가 되물음.

## 11. 비용 (현실화)

- **호스팅(권장)**: Hostinger 원클릭 Hermes VPS — **KVM2(2vCPU·8GB) ~$9/월**. 자동 주간백업·원클릭 업데이트·방화벽 포함, Telegram+Slack+Anthropic 지원, 리눅스. (대안: Bluehost/Contabo 원클릭, Fly/Railway/Render 직접배포)
- **Claude API 토큰**: 상시 루프·보고·알림·Vision의 *진짜 반복비용*. → 루틴은 저가 모델, 토큰 예산·상한 설정. (구현 계획에서 추정.)
- **문서엔진 `hwpkit`**: 무료(MIT)·순수 파이썬·리눅스 → **윈도우/한글 라이선스 불필요.**

## 13. 하지 않는 것
